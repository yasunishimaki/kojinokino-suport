# 就労ナビAI「ココマデ」 デモアプリ（プロトタイプ）

高次脳機能障害のある方の作業を支える「照合型AI」ナビゲーターの、フェーズ1（音声＋手動報告版）デモンストレーション用プロトタイプです。

**🌐 デモを触る: https://yasunishimaki.github.io/kojinokino-suport/** （Chrome推奨）

設計書: [就労ナビAIココマデ_デモアプリ設計書_v1.md](就労ナビAIココマデ_デモアプリ設計書_v1.md)

## 3つの中核体験

1. **進捗の照会** — 「今どこ?」に即答する
2. **次の一歩の提示** — 手順書と照合して「次は○○です（出典付き）」と案内する
3. **意図の再生** — 離席前の一言宣言を、あとから本人の言葉のまま返す

そして最大のポイント: **記録にないことを聞かれたら「記録がありません」と答える**（知ったかぶりをしない）。

## 使い方

ビルド不要。`index.html` をブラウザで開くだけで動きます（音声認識を使う場合は Chrome 推奨）。

```
git clone https://github.com/yasunishimaki/kojinokino-suport.git
cd kojinokino-suport
# index.html をダブルクリック、またはローカルサーバーで開く
```

### デモシナリオ（約5分）

1. **[支援員タブ]** 手順書登録に「請求書データ入力（5工程）」がプリセット済み。「▶ この手順書で開始」を押す
2. **[本人ナビ]** 「現在: 工程1 伝票の仕分け / 次: 工程2 データ入力」が表示される
3. 「✅ 終わった」ボタン（または音声「終わりました」）→ 工程2へ進む
4. デモモードでは標準所要時間が短縮されており（1分→2秒）、時間が経つと**チャイム＋音声＋画面**で「終わりましたか?」と問いかけが来る。「まだ」と答えると「了解しました。続けてください」
5. 「🚶 離席する」→「一言どうぞ」→「見積書を印刷してきます」と吹き込む（またはテキスト入力）
6. 「🔙 戻りました」のあと「何しようとしてたっけ?」と質問 →「○分前に『見積書を印刷してきます』と言っていました **(意図ログ 14:02)**」と本人の言葉＋出典で応答
7. **[信頼性の実演]** わざと「明日の予定は?」と質問 →「**記録がありません**」と答える ← デモの山場

画面下のデモバー「⏩ 10秒早送り」で問いかけタイミングを前倒しできます。

## 構成

| ファイル | 内容 |
|---|---|
| `index.html` | 本人ナビ・支援員（手順書登録/ダッシュボード/設定）・各モーダル |
| `styles.css` | 認知アクセシビリティ準拠のスタイル（大きい文字・高コントラスト・アニメーションなし） |
| `app.js` | 照合型応答エンジン・プッシュ型問いかけタイマー・意図ログ・音声入出力（Web Speech API） |

## 設計原則（全フェーズ不変）

1. 回答は「手順書・進捗ログ・意図ログ」の**記録からのみ**抽出する
2. 記録にない質問には「記録がありません」と答える（推測・一般知識での補完は禁止）
3. すべての回答に**出典**を付ける — 例: `(進捗ログ 14:02 / 手順書 工程3)`

## このプロトタイプの割り切り

- AI応答は **ローカルの照合エンジン（ルールベース）** で実装。実証版では同じ記録データを Anthropic API に渡す（設計書 5.2 のシステムプロンプト）
- データはメモリ上のみ（リロードで初期化）。実証版では Supabase へ
- 音声そのものの録音保存（テキスト化失敗時のフォールバック）は未実装。テキスト入力フォールバックで代替
- 音声認識・合成は Web Speech API を使用（Chrome 推奨。使えない環境ではボタン＋テキストで全機能が動作）
